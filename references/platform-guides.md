# 平台格式适配指南

本文档定义各平台的字数、结构、风格规范。生成内容时严格遵守对应平台的规则。

---

## 小红书笔记

### 格式规范

- **字数**：300-800 字
- **标题**：20 字以内，可用 emoji 点缀，需要抓眼球
- **正文结构**：
  - 开头 1-2 句钩子（痛点/反差/数字冲击）
  - 正文分段，每段 2-4 句，段间空行
  - 核心信息用加粗或 emoji 标记
  - 结尾互动引导（提问/号召评论）
- **话题标签**：3-5 个相关话题标签放在末尾

### 风格要点

- 口语化、亲切、有人味
- 适度使用 emoji（不过度）
- 信息密度适中，一篇只讲一个核心点
- 用「你」直接对话读者
- 避免学术化/正式化表达

### 示例结构

```
[标题] xxxxxxxx

[钩子] 1-2 句抓注意力

[正文段1] 核心观点/方法
[正文段2] 补充说明/案例
[正文段3] 实操建议

[结尾] 互动引导 + emoji

#话题1 #话题2 #话题3
```

## 微信公众号文章

### 格式规范

- **字数**：1500-3000 字
- **标题**：有冲击力但不过度标题党
- **正文结构**：
  - 开头：故事/场景/痛点引入（200-300 字）
  - 正文：用小标题分 3-5 个大段
  - 每个大段下有 2-3 个要点
  - 结尾：总结 + 升华 + 行动号召（200-300 字）

### 风格要点

- 深度优先，兼顾可读性
- 小标题引导阅读节奏
- 适当使用金句（但不过度，不要每段都有）
- 个人故事穿插增加真实感
- 结尾可以适度推广（90/10 法则）

### 示例结构

```
[标题]

[引入] 故事/场景/痛点（200-300字）

## 小标题 1
[论点1 + 支撑素材]

## 小标题 2
[论点2 + 支撑素材]

## 小标题 3
[论点3 + 支撑素材]

[总结] 核心观点回顾 + 升华 + CTA
```

## Twitter/X 推文

### 格式规范

- **字数**：280 字符以内（中文约 140 字）
- **结构**：钩子 + 正文 + 收束句三合一
- 一条推文 = 一个完整的观点

### 风格要点

- 极致精炼，每个字都有用
- 钩子在第一行，决定是否被读
- 收束句锁住互动
- 有冲击力、有观点、有立场
- 可以略微挑衅或反直觉——但要有道理

### 示例

```
营销是你能学到的最伟大的技能。
它与任何其他技能搭配。它教你人类心理学。它帮你将利基兴趣变现。
只有傻瓜才会不学把刷屏者变成客户的技能就指望赚到钱。
```

## Twitter/X 长线程

### 格式规范

- **长度**：5-15 条推文
- **首条**：最强钩子 + 「线程 🧵」标记
- **每条**：独立可读，但串联有递进逻辑
- **末条**：总结 + CTA（关注/转发/链接）

### 风格要点

- 首条决定生死——必须让人停下来
- 每条推文之间有逻辑衔接
- 可以在中间穿插个人故事
- 末条要有明确的行动号召
- 每条推文本身也要有收束感

### 示例结构

```
1/ [最强钩子] 🧵

2/ [背景/问题铺垫]

3/ [论点1 + 支撑]

4/ [论点2 + 支撑]

5/ [论点3 + 支撑]

6/ [个人故事/案例]

7/ [总结 + CTA]
```

## Newsletter（邮件通讯）

### 格式规范

- **字数**：1000-2000 字
- **结构**：金字塔原则——核心观点先行
- **一篇只深挖一个核心观点**

### 风格要点

- 像写给朋友的长信，有温度
- 个人故事穿插是标配
- 可以更深、更私密、更有观点
- 读者已经是你的订阅者，信任度更高
- 可以推到认知层级 L4-L5（产品推广区域）
- 结尾自然过渡到推广是可以的

### 示例结构

```
[主题行] 一句话概括核心观点

[开头] 个人故事/近期经历引入（200-300字）

[核心观点] 一句话总结

[论点1] + 深度展开
[论点2] + 深度展开
[论点3] + 深度展开

[总结] 回到个人故事，升华

[CTA] 自然过渡到推广（如有）

[签名]
```

## YouTube 视频脚本

### 格式规范

- **时长对应**：
  - 短视频（1-3 分钟）：300-800 字
  - 中视频（8-15 分钟）：2000-4000 字
  - 长视频（20-30 分钟）：5000-8000 字
- **结构**：钩子（前 30 秒）→ 痛点共鸣 → 3-5 个关键论点 → 总结 + CTA

### 风格要点

- 口语化——是说出来的，不是读出来的
- 前 30 秒决定留存率，必须足够强
- 可以插入「顺便说一下」式的自然推广
- 每个论点之间要有过渡
- 节奏感：长句和短句交替
- 可以直接和观众对话（「你可能在想……」）

### 示例结构

```
[钩子 - 前30秒]
一个反直觉的观点/惊人的数据/痛点场景

[痛点共鸣 - 1-2分钟]
描述观众正在经历的困境

[论点1] 为什么 + 怎么做
[论点2] 为什么 + 怎么做
[论点3] 为什么 + 怎么做

[中间推广 - 可选]
自然过渡到产品/服务

[总结]
回顾核心观点

[CTA]
点赞/订阅/评论/链接
```

## 落地页文案

### 格式规范

- **字数**：按需，通常 1000-3000 字
- **结构**：严格使用 PASTOR 框架
- **目标**：唯一目标是转化（注册/购买/预约）

### 风格要点

- 每一段都在推动读者走向行动
- 标题 = 核心价值主张（一句话说清楚产品解决什么问题）
- 社会证明不可少（数据/评价/案例）
- CTA 按钮文案要明确、有行动感
- 消除顾虑：退款保证/免费试用/FAQ

### 示例结构

```
[标题] 核心价值主张

[P - 问题] 你是否正在经历……

[A - 放大] 这个问题如何影响你的……

[S - 故事] 我也曾经……直到我发现……

[T - 见证] XX 位用户已经通过这个方法……
  - 用户评价 1
  - 用户评价 2
  - 数据证明

[O - 产品] 介绍产品/服务
  - 包含什么
  - 核心收益
  - 价格

[R - 行动号召] 立即开始 / 限时优惠

[FAQ] 常见问题解答
```

## 通用原则（所有平台适用）

1. **钩子永远是第一优先级**：无论什么平台，前 1-2 句/前 30 秒决定成败
2. **一篇一个核心点**：不要试图在一篇内容里塞太多东西
3. **清晰胜于聪明**：不卖弄文采，确保读者能理解你在说什么
4. **考虑分享性**：如果你自己不会分享这篇内容，别指望别人会
5. **读者视角审视**：写完后从读者角度重新阅读一遍，问自己「我会读完吗？」
