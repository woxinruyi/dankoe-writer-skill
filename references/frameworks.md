# 写作框架详解

本文档包含 Dan Koe 方法论中所有写作框架的详细说明、结构模板和示例。

---

## 1. 金字塔原则（元框架）

所有长内容的底层结构，适用于任何写作类型。

### 结构

```
核心观点（结论先行）
├── 论点 1
│   └── 支撑素材（事实/数据/故事/引用）
├── 论点 2
│   └── 支撑素材
└── 论点 3
    └── 支撑素材
```

### 要点

- 读者一开始就知道这篇内容是否与自己相关
- 读者能跟随逻辑走到结论
- 简单、可复制
- 需要一个「资料库」保存高亮、推文、研究、引用

### 示例结构

```
[核心观点] 说服力是 21 世纪最重要的技能。
  [论点1] 你每天都在说服别人——只是无意识地。
  [论点2] 有意识的说服 = 互利，而非自利。
  [论点3] 不掌握说服力，你的好东西也无人知晓。
  [每个论点下] 用个人经历 + 书籍引用 + 数据支撑
```

## 2. P&P 框架（痛点 + 过程）

最简单的框架，适合推文、短帖、小红书笔记。

### 结构

```
痛点/问题（1-2 句）
↓
解决步骤（3-6 个要点）
↓
收束句（总结/升华/行动号召）
```

### 示例

```
如果你总是疲惫不堪，那就：
- 半夜别刷手机
- 修复糟糕的饮食
- 去散步
- 动起来
- 迷上一个新兴趣
你之所以疲惫，是因为你被某种重复的生活困住。
```

### 进阶技巧

- 「过程」越独特，内容越强
- 给过程起名字（如「一人公司」「4 小时工作日」），创造权威标签
- 收束句锁住互动，不要死板套框架

## 3. PASO 框架（问题-放大-解决-指向）

营销型短内容的核心框架。

### 结构

```
P - Problem（问题）：指出读者正在经历的问题
A - Agitate（放大）：描述问题如何影响生活/关系/工作
S - Solution（解决方案）：给出解决方案
O - Offer（指向）：如果有产品/服务，自然过渡
```

### 示例

```
[P] 你写了很多内容，但粉丝就是不涨。
[A] 你花了几个月的时间，每天发帖，但互动寥寥。
    你开始怀疑是不是自己不适合做内容，想放弃。
    但问题不是你——是你的方法。
[S] 你需要的不是更多内容，而是经过验证的内容结构。
    研究表现最好的帖子，拆解它们的结构，然后复刻。
[O] 我在 [产品名] 里整理了 50 个验证过的模板，链接在简介。
```

### 要点

- 放大痛点时，用个人经历最有说服力
- O（指向）不是硬广，而是自然延伸
- 干货型可以省略 O，变成 PAS

## 4. PASTOR 框架（问题-放大-故事-见证-产品-行动号召）

营销型长内容的完整框架，适合 Newsletter、落地页、公众号长文。

### 结构

```
P - Problem（问题）：描述读者面临的问题
A - Agitate（放大）：放大问题的痛苦和影响
S - Story（故事）：讲一个相关的转变故事（最好是自己的）
T - Testimony（见证）：社会证明——他人的结果/评价
O - Offer（产品）：你的解决方案/产品
R - Response（行动号召）：明确告诉读者下一步做什么
```

### 示例骨架

```
[P] 你想做自己热爱的工作，但不知道从何开始。
[A] 你试过很多副业，结果都不了了之。你看着别人在网上赚钱，
    自己却连第一步都迈不出。时间一天天过去，焦虑越来越重。
[S] 三年前，我也是这样。工作让我窒息，试了 7 种生意都失败了，
    欠了 8000 美元。直到我发现了一个简单的模式：
    写作 → 建受众 → 产品化。第一年赚了 10 万。
[T] 已经有 500+ 学员用这个方法从零开始建立了自己的在线收入。
[O] 我把整个方法论做成了 [课程名]，包含写作、营销、产品创建
    的完整体系。
[R] 点击下方链接了解详情。本周价格提升前最后机会。
```

## 5. PAS 框架（问题-放大-解决方案）

推广帖专用，PASO 的简化版。

### 结构

```
P - Problem（问题）：提出问题吸引注意
A - Agitate（放大）：放大问题的影响
S - Solution（解决方案）：你的产品/服务是答案
```

### 使用场景

- 帖子评论区的推广
- Instagram/小红书故事的推广
- 正文中自然植入的推广段落

### 示例

```
[P] 如果你正在为「不知道写什么」而发愁——
[A] 这个问题不解决，你会一直在「想写但写不出」的循环里打转，
    看着别人越来越好，自己越来越焦虑。
[S] 我做了一个 [产品名]，7 天内帮你打通写作卡点。链接在简介。
```

## 6. 可操作原则型

建立权威的干货型框架。

### 结构

```
钩子（1-2 句，引出话题）
↓
原则/步骤列表（3-7 个要点，每个可操作）
↓
大观点总结（收束句，传达核心信息）
```

### 示例

```
你的内容很糟糕，因为你没有：
- 钩住读者
- 激起他们的问题
- 与他们的痛点产生共鸣
- 给他们一个有价值的解决方案
人类心理学总是会打败你认为会有效的东西。清晰而不是聪明。
```

### 要点

- 帮助在特定兴趣中建立权威
- 初学者可以理解和使用的原则
- 钩子 → 正文 → 结论不可省略

## 7. 重要性阐述型

改变行为认知的干货型框架。

### 结构

```
钩子（引出话题）
↓
为什么重要（3+ 个理由）
↓
行动号召/收束句
```

### 示例

```
营销是你能学到的最伟大的技能。为什么？
- 它与任何其他技能搭配
- 它教你人类心理学
- 它帮助你将利基兴趣变现
只有傻瓜才会期望不学习把刷屏者变成客户的技能就能赚到钱。
```

### 要点

- 关键在于找到正确的「为什么」
- 真正改变行为的是「为什么」，不是「怎么做」
- 可以抛出曲线球——意外的理由更能抓住注意力

## 8. 常见错误型

建立权威、帮助读者避坑的干货型框架。

### 结构

```
指出错误（1-2 句）
↓
为什么这是错的（解释影响）
↓
正确做法（给出替代方案）
```

### 示例

```
作为初学者停止写超特定的内容。
你应该专注于增长。让你的写作有共鸣。
专注于一个共同的问题。从高层次谈论这个话题。
如果只有少数人会分享你的内容，就不要指望增长。
```

### 要点

- 表明「我知道他在说什么，因为他能指出问题」
- 任何兴趣都有常见错误可以写
- 结合个人犯过的错误效果更好

## 9. 五层认知模型 × 内容策略

不同认知层级的读者需要不同类型的内容。

### 各层级内容策略

| 层级 | 读者状态 | 适用框架 | 内容策略 | 发布渠道 |
|------|---------|---------|---------|---------|
| L1 无意识 | 不知道问题存在 | P&P / 常见错误型 | 用痛点唤醒，广泛话题 | 社交媒体 |
| L2 问题意识 | 知道问题不知道解法 | 重要性阐述型 | 放大问题影响，激发紧迫感 | 社交媒体 |
| L3 解决方案意识 | 知道有解法 | 可操作原则型 | 给可执行建议和步骤 | 社交媒体 / Newsletter |
| L4 产品意识 | 知道你的方案 | PASO / PASTOR | 突出独特方法和差异化 | Newsletter / 落地页 |
| L5 高度意识 | 准备行动 | PAS | 消除最后顾虑，给行动理由 | 落地页 / 私信 |

### 核心原则

- 社交媒体内容主攻 L1-L3（教育、唤醒、提供价值）
- Newsletter / 产品 / 落地页推到 L4-L5（说服、转化）
- 所有框架从「痛点→解决」开始，就是为了把读者从低层往高层推
- 不同层级的读者，同一个信息要用不同的表达方式

## 10. 信任矩阵（辅助框架）

不是具体的写作结构，而是评估每篇内容是否平衡的标尺。

### 三根支柱

| 支柱 | 定义 | 如何实现 |
|------|------|---------|
| **增长** | 内容是否有传播力，能否被分享 | 用验证过的结构、广泛共鸣的话题、有冲击力的表达 |
| **真实性** | 是否表达了你的核心信念和个人故事 | 融入个人经历、独特观点、与众不同的视角 |
| **权威性** | 是否展示了专业能力 | 可操作建议、数据支撑、深度分析、案例拆解 |

### 使用方式

每篇内容至少兼顾两根支柱。如果三根全覆盖，内容质量通常最高。
- 缺增长 → 内容深但没人看 → 拓宽话题、优化钩子
- 缺真实性 → 像搜索引擎一样没人味 → 加入故事和个人视角
- 缺权威性 → 有趣但不可信 → 加入数据、方法论、深度分析
